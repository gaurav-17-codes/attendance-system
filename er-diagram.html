⚡ Key Database Constraints

User.email @unique

No two users can share the same email address

StudentClass @@unique([studentId, classId])

A student can only be enrolled in a class once

Attendance @@unique([studentId, classId, date])

One attendance record per student per class per day — prevents duplicates

Attendance.date @db.Date

Stores date only (no time) — prevents timezone-related duplicate bugs

StudentClass onDelete: Cascade

Deleting a student removes all their enrollments automatically

Class isActive: Boolean

Soft delete — deactivating a class preserves all historical attendance data
